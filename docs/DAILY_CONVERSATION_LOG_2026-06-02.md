## 요청 원장

### 1. Pivot/분류프로그램/출고분배 업로드 구조

- 사용자는 `nenova.exe` Pivot에서 통합 엑셀을 다운받고, 분류프로그램으로 결과를 만든 뒤, 그 결과를 수정한다.
- 수정된 수량을 출고분배에 입력해야 한다.
- 엑셀 수정 후 업로드하면 출고분배가 자동 적용되기를 원한다.
- 시각적으로 엑셀을 보여주고, 읽기/검증 과정을 로그로 확인하고 싶어 했다.
- 업체의 품목별 총수량 기준으로 주문등록값과 실제 분배값 차이를 확인하고, 사용자가 100% 매칭이라고 판단하면 업로드 적용되게 해야 한다.
- 반복적으로 문제 삼았던 분배 오류 때문에 `nenova.exe` 설정값과 동일한 방식으로 작업되어야 한다.

### 5. 챗봇 기능과 자연어 답변

- 지금 챗봇이 답변 가능한 질문들을 정리해달라고 요청.
- Claude API를 쓰는데도 자연어 대화로 답변받고 작업하는 구조가 이루어지지 않는 이유를 물었다.
- `네노바 챗봇에서 22-2차 콜롬비아 장미 농장 수량` 질문 결과가 원하는 값이 아니었다.
- 원하는 답변 형태는 `품목 어디농장 10단 어디농장 10단`처럼 품목과 농장별 수량이 나오는 구조.
- `지금 너랑 대화하듯이 챗봇도 대화가 되어지지 않는 이유`를 물었고, 추가해달라고 요청했다.
- 최근 챗봇 감사 로그 기준으로, API/네트워크 오류보다 “성공 응답인데 의도나 업체 매칭이 틀린” 문제가 핵심이라고 정리했다.
- 예: `콜롬비아 장미 농장`을 거래처 콜롬비아로 보는 문제, 분배수량을 출고분배가 아닌 다른 조회로 돌리는 문제.
- `20-1차 어떻게 보여드릴까요` 응답에는 가능한 표시 방식 예시까지 같이 보여주길 원했다.

### 6. 재고 기준 통일

- `22-1차 태국`에서 입고와 출고는 맞는데 재고 수량이 문제 생기는 현상이 있었다고 했다.
- `nenova.exe` 재고관리에서 실제 인보이스와 다른 입고 수량을 `+/-`로 조정하는 경우가 있으므로, 챗봇 재고 답변과 네노바웹 차수피벗 표시도 이 수동재고조정 기준을 반영해야 한다고 요청.
- 재고 질문은 차수확정 후 재고가 아니라, 확정을 항상 해놓을 수 없으므로 `전재고 + 입고 + 수동재고조정 - 출고` 기준 잔량으로 답해야 한다.
- 수량 0 제외 기준으로 답변해야 한다.
- 답변 형식은 `콜롬비아 수국` 아래 품목/수량, `콜롬비아 장미` 아래 품목/수량처럼 그룹화해야 한다.
- 사용자가 챗봇에게 “0 제외 후 답변”을 요청했더니 네트워크 오류가 발생했다고 했다.
- 정리 방식도 사용자가 챗봇에게 말하면 답변 구조가 수정되는 방식이 가능한지 물었다.

### 7. 붙여넣기 주문등록 흐름

- 붙여넣기 주문등록에서 일괄 등록+분배 2건 실행했는데 주문등록만 되고 분배가 안 된다고 보고.
- 이후 주문등록도 되었고 분배도 되었다고 나오지만, `nenova.exe` 출고분배 화면에는 표시가 안 뜨는 문제를 보고.
- 기존 작업내역을 찾아 삭제해달라고 요청. `nenova.exe`에서는 수정할 수 없는 값이라고 설명했다.
- 분배수량을 0으로 했을 때 주문등록 삭제 시도가 안 되는 문제 보고.
- `붙여넣기 주문등록시 Invalid column name 'OrderYearWeek'` 오류 보고.
- `Violation of PRIMARY KEY constraint 'OrderDetail_PK'` 중복키 오류 보고.
- `Transaction ... was deadlocked ... deadlock victim` 오류 보고.
- 붙여넣기 주문등록에서 `추가`/`취소` 규칙 재정의:
  - `추가`, `취소` 단어가 없으면 기본 `추가`.
  - `취소`가 있으면 주문등록 음수 변경.
- 매칭 후 단위/수량이 나오는데, 같은 품종의 단위를 일괄 변경할 수 있어야 한다.
- 네덜란드 품종에 한해서 품목관리 비고의 MOQ 값이 주문등록 비고에 표시되길 원했다.
- 주문즐겨찾기:
  - 기존 차수별 주문등록 중 선택하면 품목/수량이 즐겨찾기로 저장.
  - 차수 선택 후 주문등록하기를 누르면 선택 차수에 등록.
  - 품목수량은 수정 저장 가능해야 한다.
  - 나중에는 붙여넣기 주문등록 페이지 상단에 큰 `주문즐겨찾기` 버튼 하나가 있고, 누르면 큰 새창에서 모든 작업을 하길 원했다.

### 8. 차수피벗/Pivot 통계/물량표 다운로드

- 주문등록 메뉴의 주문 품목정보가 모든 품목에 다 나열되어 있는데, `nenova.exe`처럼 묶음으로 보여야 한다고 요청.
- `nenova.exe` Pivot 통계 메뉴와 `nenovaweb` Pivot 통계 메뉴가 기능이 같아야 한다고 했다.
- Pivot 통계를 갖고 통합 물량표를 다운받는데, “물량표 다운받기” 기능이 있어 차수 선택만으로 분류프로그램 없이 다운받을 수 있는지 물었다.
- 화면 이미지 기준으로 웹과 exe의 펼치기 기준이 다르다고 지적.
- 물량표 다운받기 버튼이 안 보인다고 했고, 배포가 안 된 것 아니냐고 물었으며 배포 요청.

### 9. 물량표 엑셀 형식/다운로드 요구사항

- 물량표 엑셀에서 수국과 루스커스는 하나로 합쳐야 한다.
- 알스트로는 수량을 16으로 나누어야 한다.
- 품목명에 들어가는 `rose`, `carnation`, `hydrangea`, `alstromeria` 같은 단어와 `/` 포함 텍스트는 제거해야 한다.
- 숫자는 가운데 정렬.
- 업체 셀 너비는 2글자 기준 균일화.
- 농장 이름은 한글화: `ayura` → `아유라`, `colibri` → `콜리브리`, `don eusebio` → `돈유세비오` 등.
- `#VALUE!`가 뜨는 이유와 숫자 뒤에 점이 찍히는 이유를 물었다.
- 업체 수가 10개 이하일 때 중간에 품명 더 넣는 것을 없애길 원했다.
- 업체 셀 너비가 2글자가 아니라 3개 단어 기준처럼 보이는 문제를 지적.
- 엑셀 시트 분류는 콜롬비아 외에도 태국/호주/네덜란드는 각각 묶어야 한다.
- 중국은 중국기타와 중국장미를 합쳐야 한다.
- 네덜란드 품목에 한해서 품목 옆 셀에 한글 컬러가 들어가길 원했다.
- 네덜란드의 `/` 앞 텍스트는 업체명이 아니라 컬러 이름으로 적용되어야 한다.

### 10. 엑셀 업로드 검증/적용 흐름

- 물량표 수정 후 업로드하는 방법을 물었다.
- 승인 후 업로드도 주문등록/분배 수정이므로 이전 오류가 재발할 수 있는지 검증했는지 물었다.
- 검증할 때 품목/수량 비교를 차수피벗 엑셀시트처럼 변경사항을 볼 수 있게 하길 원했다.
- 업로드 검증 기준은 분배기준이 아니라 기존 주문등록 기준이어야 한다고 명확히 했다.
- 최초 분배 후 수정이 아니라, 주문등록된 기준으로 분배하는 작업이기 때문이다.
- 화면에 몇 건만 나오는데 실제로는 많아야 하며, 업체 순서도 기존 순서와 같아야 보기 쉽다고 했다.
- 검증 후 일괄 주문등록 신규추가 및 분배가 되어야 한다.
- 분배할 때 차수확정처럼 로그 표시가 되고, 업체명/품목명 중심의 직관적인 작업 현황을 눈으로 확인하고 싶다고 했다.
- `nenova.exe`에서 일괄 분배해 놓고 변경사항만 수정/추가 등록/분배하면 오류와 시간이 줄어드는지 구조 확인 요청.
- 적용을 눌렀는데 로그창이 안 뜨고 적용완료만 뜬다고 보고.
- 작업 완료 후 출고일 지정이 안 되어 출고일과 출고량이 안 맞을 수밖에 없다고 지적.
- 예: 공주플라워 `Mariposa`가 `0>1`로 변경됐는데 출고일 분배가 안 되어 출고일 없는 상태.
- 동일 문제 있는 행을 찾아 수정 요청.
- `Multiple columns are specified in an aggregated expression containing an outer reference...` SQL 오류 보고.
- 출고분배 엑셀 업로드 기본 차수는 기존 현재 차수 계산에서 +1이어야 한다고 요청. 예: 22가 아니라 23.
- `2301장미물량표.xlsx` 검증에서 `FREEDOM 50` 같은 경우 분배 틀린 것이 많은데 검증되지 않는다고 지적.
- 엑셀 업로드 적용 중 작업 현황 새창이 안 뜬다고 보고.
- `2301수국물량표.xlsx`에서 물량표에선 `여분코드`를 지웠는데 업로드 분석에는 나오지 않는다고 지적.

### 11. 23차 대량 작업 불안과 전체 재검증

- 사용자는 “이번 자동분배에서 문제 생기면 양이 너무 많아서 롤백도 힘들다”고 강하게 우려했다.
- `NENOVA.EXE`, SQL, `NENOVAWEB`에서 문제 생길 것 없는지 전체 작업 재확인 요청.
- 작업이 끝나면 오늘 작업한 내용 중 데이터값 수정 시 `nenova.exe`에서 오류 발생할 이슈를 기존 이슈와 비교해서 찾아내고, `nenovaweb`에서도 오류가 생기지 않게 검증해달라고 요청.
- `nenova.exe` 오류 MD를 확인하고 충돌 구조가 있는지, 견적서/주문등록/분배 모두 문제 없는지 확인 요청.
- 붙여넣기 주문등록도 MD 확인 후 다시 검증 요청.
- `nenovaweb`의 모든 기능 버튼 재점검 요청.

### 12. 차수피벗 업체별 출고리스트/단가 표시

- 차수피벗에서 엑셀 다운로드 시 토탈시트와 업체별 시트가 만들어지는데, 품목/수량 합계에 품목단가, 합계단가, 총금액도 넣어달라고 요청.
- 업체별 품목 단가를 포함할 수 있는지 물었다.
- 이미지처럼 차수, 업체이름, 분류를 차수피벗 엑셀다운로드에 표시되게 요청.
- 수량 변경 후 업로드 방법도 설명해달라고 했다.
- 수량 변경되는 부분은 주문등록, 분배, 출고일 지정 등 계속 오류가 있었으므로 동일 오류 재발 없이 구성해야 한다고 강조.

## 보완된 작업내역 관리 기준

앞으로 긴 세션 또는 운영 위험 작업은 4단계로 기록한다.

### 1. 일일 대화 원장

- 파일명: `docs/DAILY_CONVERSATION_LOG_YYYY-MM-DD.md`
- 포함 항목:
  - 사용자 요청 원장
  - 언급 파일
  - 업무 의도
  - 핵심 결정
  - 오늘 커밋 목록
  - 검증/배포 결과
  - 남은 위험
- 사용자가 “오늘 대화 저장”, “작업내역 저장”, “MD 저장”을 말하면 반드시 생성/갱신한다.

### 2. 기능/사고별 상세 문서

- 파일명 예:
  - `docs/SHIPMENT_IMPORT_DATE_BASE_OUTDAY_FIX_2026-06-02.md`
  - `docs/ERP_WRITE_CONFLICT_AUDIT_2026-06-02.md`
- 포함 항목:
  - 증상
  - 대표 사례
  - 원인
  - 보정 SQL/코드
  - 검증 결과
  - 재발 방지
  - 관련 커밋

### 3. 전체 이력 인덱스

- `docs/work_history.md`에는 상세문서 전체를 복붙하지 않고, 날짜별 링크와 핵심 요약만 남긴다.
- 긴 대화는 반드시 일일 대화 원장에 링크한다.

### 4. 작업 시작 전 검색 체크
