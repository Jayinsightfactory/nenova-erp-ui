# 24-01 exe ↔ 웹 정합 점검 (단위 환산 포함) — 2026-06-10

> 대상 차수: **24-01** · 도구: `scripts/compare-week-exe.js` (`npm run test:compare-week -- 24-01`)
> 목적: nenova.exe(전산) 가 만든 출고/견적과 웹 계산이 일치하는지, 특히 **단→박스 10배(단위 환산)** 틀어짐이 24-01 에 남아 있는지 자동 비교.

---

## 1. 점검 항목 (compare-week-exe.js 5개 섹션)

| # | 섹션 | 호출 엔드포인트 | 무엇을 보나 |
|---|------|----------------|-------------|
| [1] | 견적 불변조건 | `GET /api/estimate/validate?week=24-01` | 견적 불변식(금액식·환산) pass/fail |
| [2] | 출고일/Est 진단 | `GET /api/shipment/estimate-period-repair?weeks=24-01` | ShipmentDtm·EstQuantity 불일치 건수 |
| [3] | Detail↔Date↔View | `GET /api/dev/estimate-cost-source-audit?week=24-01` | Cost/Amount/Vat 소스 3중 불일치 |
| [4] | 웹 견적 금액식 | `GET /api/estimate?shipmentKey=…&byDate=1` | 단가×수량 = Amount+Vat 검증 |
| **[5]** | **단위 환산 정합 (신규)** | `GET /api/shipment/distribute?type=products&week=24-01` | **출고합/주문합 ≈ BunchOf1Box → 단→박스 10배 의심** |

### 신규 [5] 단위 환산 정합 — 판정식

품목별로 `distribute?type=products` 가 주는 집계값을 사용:

```
조건: OutUnit = '박스'  AND  BunchOf1Box(B1B) > 1  AND  orderQty > 0  AND  outQty > 0
비율: ratio = outQty(출고합) / orderQty(주문합)
의심: B1B × 0.85 ≤ ratio ≤ B1B × 1.15   → "단(묶음)을 박스로 저장한 10배" 의심
```

- 장미 B1B=10 → 비율 ≈ 10, 카네이션 B1B=15 → 비율 ≈ 15 일 때 경고.
- `probe-unit-mismatch.js` 의 주문-기준선(order baseline) 판정과 동일한 신호를 **운영 API 만으로** 재현한 것 → exe↔웹 cross-reference 가능.

### [5] 의 한계 (문서화된 사각지대)

- **주문과 출고가 둘 다 10배**인 경우(주광 단독 재업로드: DB 주문 50 / 출고 50) ratio ≈ 1 이라 [5] 로는 안 잡힌다.
  → 이 케이스는 **다른 차수/동료 중앙값** 비교가 필요: `scripts/probe-import-qty-2401.js 24-01 주광`.
- [5] 는 품목 **집계(sum)** 비율이라, 일부 업체만 10배면 평균이 희석될 수 있음 → 업체 단위 정밀 확인은 `probe-unit-mismatch.js`(거래처×품목 행 단위).

## 3. 의심 발견 시 후속 (remediation)

```bash
# 1) 행 단위 정밀 진단 (읽기 전용)
node scripts/probe-unit-mismatch.js 24-01
node scripts/probe-import-qty-2401.js 24-01 주광     # 주문·출고 둘 다 10배 케이스

# 2) 보정 dry-run (DB 무수정, before/after 표 확인)
npm run repair:unit-mismatch -- --week=24-01 --cust=주광 --limit=50

# 3) 확인 후 실제 보정 (트랜잭션, --apply 필요)
npm run repair:unit-mismatch -- --week=24-01 --cust=주광 --limit=50 --apply
```

`repair-unit-mismatch.js` 는 주문 baseline 으로 확정된 행만 `OutQuantity / BunchOf1Box` 로 되돌리고
나머지 환산필드는 canonical `distributeUnits` 로 재계산한다(13/14차 단일 공식 유지). 자세한 내용은
`docs/OUTUNIT_WRITE_AUDIT_2026-06-10.md` 참조.
