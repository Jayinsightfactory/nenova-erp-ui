# 작업/대화 기록 유지 원칙 (2026-05-25)

## 사용자 지시

대화기록과 작업기록은 반드시 남겨야 한다.

## 적용 원칙

앞으로 작업을 진행할 때 다음 기록을 남긴다.

1. Git 커밋
   - 실제 코드 변경 단위별로 커밋 메시지를 남긴다.

2. 작업 기록 문서
   - 기능 수정, 운영 검증, DB 진단, 배포 확인이 있는 작업은 `docs/` 아래 별도 문서로 남긴다.
   - 문서에는 사용자 요청, 확인한 구조, 수정 내용, 검증 결과, 남은 판단 지점을 포함한다.

3. 대화 의도 기록
   - 사용자가 왜 해당 기능을 요청했는지 기록한다.
   - 단순 구현 내용만 적지 않고 업무 목적도 같이 적는다.

4. 배포 확인 기록
   - 배포한 경우 운영 `git-log` 확인 결과와 커밋 해시를 문서에 남긴다.
