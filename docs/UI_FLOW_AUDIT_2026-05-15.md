# NenovaWeb UI Flow Audit - 2026-05-15

## 이번에 확인한 흐름

1. 주문등록 새창/팝업에서 품목 그룹을 선택하고 수량을 입력하는 흐름
2. 붙여넣기 주문등록 후 차수피벗/엑셀 화면으로 이동하는 흐름
3. 차수피벗 및 통계 피벗에서 차수 범위를 이동하는 버튼 흐름

## 확인된 문제와 조치

| 구분 | 확인된 문제 | 조치 |
|---|---|---|
| 주문등록 그룹 선택 | 그룹 클릭 함수가 선택 그룹 상태를 직접 갱신하지 않아 팝업/새창에서 선택 상태와 표시 목록이 어긋날 수 있음 | `loadGroupProds`가 선택 그룹과 품목 목록을 함께 관리하도록 수정 |
| 주문등록 그룹 합계 | 왼쪽 그룹별 주문수량이 현재 로드된 품목 목록 기준으로만 계산됨 | 전체 `quantities` 기준으로 그룹별 합계를 계산하도록 수정 |
| 기존 주문 조회 | `counName` 저장식 우선순위가 잘못되어 품목 그룹 매칭 정보가 비어질 수 있음 | `item.counName || ''`로 명확히 저장 |
| 신규 버튼 | 신규 초기화 후 이전 품목 목록이 남을 수 있음 | 선택 그룹과 품목 목록을 같이 초기화 |
| 붙여넣기 주문등록 -> 피벗 | 차수피벗 이동 버튼이 같은 창 이동이라 등록 화면이 사라짐 | 차수 조건을 가진 차수피벗을 새 창으로 열고, 팝업 차단 시 같은 창 이동 fallback |
| 통계 피벗 차수 이동 | 시작/종료 차수를 같이 움직이는 범위 이동 버튼이 부족함 | 범위 전체 차수/주차 이동 버튼 추가 |

## 아직 계속 점검할 항목

1. Nenova.exe의 주문등록 화면에서 그룹 선택, 품목 표시, 합계 표시가 웹과 100% 동일한지 화면 단위 비교
2. 차수피벗의 엑셀 다운로드 컬럼 순서, 잔량복사 값, 필터 상태 유지 방식 비교
3. 출고분배 화면에서 주문등록 직후 자동 반영되는 시점과 히스토리 기록 방식 반복 검증
4. 경영지원/Ecount ERP 화면의 원본 데이터 컬럼과 웹 계산값 비교표 작성
