# 영업수입불량차감 품목·국가 매칭 대조 결과

기준일: 2026-07-22 (운영 DB read-only 조회)

## 확인한 실제 데이터

| 항목 | 결과 |
|---|---:|
| Product 전체 행 | 3,230 |
| 삭제 제외 활성 Product | 3,088 |
| 활성 Product 중 DisplayName 미입력 | 3,086 |
| 활성 Product 중 CounName 미입력 | 0 |
| 활성 Product 중 FlowerName 미입력 | 0 |
| 콜롬비아 장미 | 369 |
| 콜롬비아 카네이션 | 306 |
| 중국 장미 | 188 |
| 중국 카네이션 | 80 |
| 붙여넣기 주문 매칭 캐시(order-mappings.json) | 332건 |

`DisplayName`은 거의 비어 있으므로 화면 검색을 DisplayName만으로 처리하면 한글 입력이 영문 `ProdName` 품목을 찾지 못한다. 실제 전산 기준은 `Product.ProdKey → Product.CounName/FlowerName/ProdName`이며, `CountryFlower`는 국가와 품종을 합친 표시용 값이지 국가 필드가 아니다.

대표적으로 다음 품목은 모두 같은 카네이션 품종군이지만 국가가 다르다.

| ProdKey | ProdName | CounName | FlowerName |
|---:|---|---|---|
| 417 | CARNATION Kaori | 콜롬비아 | 카네이션 |
| 447 | CARNATION Moon Light | 콜롬비아 | 카네이션 |
| 456 | CARNATION Novia | 콜롬비아 | 카네이션 |

현재 영업수입불량차감 저장행은 `ProdKey`로 Product를 다시 조인했을 때 국가 누락 0건이었다. 따라서 국가가 사라지는 원인은 DB 국가값이 아니라 표시/엑셀 그룹화 단계에 있었다.

## 원인과 수정

1. 엑셀 그룹 키가 `품종`만 사용해 같은 업체의 콜롬비아 장미와 중국 장미가 한 그룹으로 합쳐질 수 있었다. 이제 `국가 + 품종`을 그룹 키로 사용해 `콜롬비아 장미`, `중국 장미`를 각각 표시한다.
2. 저장행의 국가 표시를 `Product.CounName` 조인값으로 고정했다. `CountryFlower`를 국가로 오인하지 않는다.
3. 한글 검색 시 SQL 원문 후보만 사용하면 `문라이트`가 들어간 중국 행만 남고 `CARNATION Moon Light`가 제외됐다. 한글 검색은 전체 활성 Product를 번역/자모/영문 별칭 기준으로 재평가한다.
4. 후보 정렬이 문자열 점수만 우선해 중국의 한글 표기 후보가 먼저 나올 수 있었다. 이제 명시 국가를 최우선으로 제한하고, 국가 미지정 장미·카네이션은 콜롬비아 운영 기본값을 보조하며, 실제 `OrderDetail` 사용량과 `order-mappings.json` 붙여넣기 매칭 횟수를 함께 반영한다.
5. `문라이트` 실제 사용량은 ProdKey 447(콜롬비아 카네이션)가 2,298회, 중국 후보는 2회 이하였고, 수정 후 검색 1순위가 447로 확인됐다. `중국 문라이트`를 명시하면 중국 후보가 1순위로 유지된다.
6. 매칭 이력은 기본 펼침으로 표시하고, 기존 이력의 JSON도 읽어 과거 `ProdKey/CustKey` 매칭 변경을 별도 목록으로 표시한다.
7. 빈 행 추가 영역은 행이 있을 때도 표 마지막에 항상 남긴다. 추가한 빈 행은 기존 행 아래에 append된다.

## 기존 붙여넣기 주문등록과의 관계

붙여넣기 주문등록의 `displayName.js` 한글↔영문 사전과 `order-mappings.json`을 후보 검색에 재사용했다. 다만 불량차감은 기존 계약대로 후보를 자동으로 확정하지 않고 사용자가 후보를 선택해야 `ProdKey`를 저장한다. 견적서 `Estimate` 쓰기는 기존과 같이 “견적서관리 등록”을 명시적으로 실행할 때만 발생한다.

국가 미지정 입력의 콜롬비아 우선은 후보 순서 규칙이며, 사용자가 `CHINA/중국`을 입력하면 중국으로 필터링한다. 국가가 DB에 없는 상태에서 문자열로 국가를 추정해 저장하지 않는다.

## 엑셀 출력

- 품명 F열 폭을 원본 15에서 37.5로 확대했다(2.5배).
- 업체·품종 중복 표시는 기존 양식대로 공백 처리한다.
- 업체 사이 빈 행을 유지한다.
- 품종 표시에는 `Product.CounName + Product.FlowerName`을 사용한다.
- 품명/국가가 다른 행을 한 품종으로 합치지 않는다.

