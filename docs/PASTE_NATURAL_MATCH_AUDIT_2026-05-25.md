# 붙여넣기 주문 자연어 매칭 점검 (2026-05-25)

## 점검 기준
- `data/order-mappings.json`의 저장 매핑 487건을 자연어 입력명과 매핑된 품목명으로 비교했다.
- 카카오톡 원문 `카톡.txt`에서 `추가/취소/박스/단/송이`가 들어간 품목 후보를 추출해 빈도가 높은 자연어 표현을 확인했다.
- 운영 Product 전체 API는 현재 응답 지연이 있어, 이번 작업은 저장 매핑과 카톡 원문, 런타임 DB 매칭 로직의 별칭 사전 보강 중심으로 진행했다.

## 확인된 주요 리스크
- 호주 외에도 카네이션, 장미, 알스트로, 수국 색상, 레몬잎/솔리다고 소재류에서 자연어 표현이 많다.
- 자동 저장 매핑 중 일부는 특정 품목으로 과하게 몰려 있다.
  - 예: 중국 장미 주밀리아, 콜롬비아 수국 HYDRANGEA, 알스트로 Arrone, 폼폼이 화이트.
- 이 과거 자동 매핑은 현재 `parse-paste`에서 `auto=true`이고 과밀 prodKey이면 그대로 신뢰하지 않도록 이미 우회한다.
- 레몬잎처럼 실제 품목명과 운송료/항공료 품목명이 같이 검색될 수 있는 경우가 있어 비용성 품목 제외 조건이 필요했다.

## 이번 보완
- 자연어 별칭을 추가했다.
  - 카네이션: 베로나핑크, 피치맘보, 로다스크림, 딜레타크림/옐로우, 고블린, 엠마, SP 계열.
  - 장미: 플라야블랑카, 프리덤, 프라우드/프라우드레드, 쉬머, 오션송, 하츠, 비스윗, 크림컵, 버터컵, 안슬레이, 로맨틱비치, 다이아나.
  - 알스트로/색상: 오렌지퀸/오랜지퀸, 라벤더/라벤다, 연그린/진그린/진핑크.
  - 소재: 레몬잎, 솔리다고, 호주 엄블렐라/울리브러시/고아나 클로우 오타.
- 카톡식 섹션 헤더에서 `호주`, `레몬잎`, `루스커스`, `호접`, `덴파레`, `리시안`, `튤립` 문맥도 읽도록 확장했다.
- 고객명이 없는 `16차 호주 추가` 같은 섹션형 입력은 기본 `여분코드` 주문으로 파싱 가능하게 했다.
- 입력에 운송료/항공료가 명시되지 않으면 `운송료`, `freight`, `shipping`, `charge` 품목은 후보에서 제외한다.

## 남은 확인
- 운영 Product 전체 목록 API가 정상 응답할 때 전체 DB 품목명 기준으로 재감사하면 더 정확하게 남은 사각지대를 줄일 수 있다.
- 특히 저장 매핑에 없는 품목은 이번 별칭 사전으로 런타임 DB 후보 점수가 올라가지만, 실제 prodKey 확정은 운영 DB 품목명이 필요하다.
