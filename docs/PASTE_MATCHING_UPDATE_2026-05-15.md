# 붙여넣기 주문등록 매칭 개선 기록

## 기준 자료

- `C:\Users\USER\Documents\카카오톡 받은 파일\카톡.txt`
- 주요 패턴: `업체 - 품목 3박스 취소`, `업체 : 품목 2박스 추가`, 업체명 단독 줄 다음 품목 추가/취소 줄, `변경사항` 섹션 헤더.

## 개선 내용

- 품목 매칭 키에서 수량, 단위, 추가/취소/변경/출고 같은 변동 단어를 제거한다.
- 로컬 브라우저 매칭 캐시가 정확 일치뿐 아니라 토큰 기반 부분 일치로 재사용된다.
- 수동 품목 선택 시 로컬 캐시와 서버 공용 매핑을 같은 방식으로 저장한다.
- 주문 저장 성공 시 취소 품목까지 매핑 학습 대상에 포함한다.
- 일괄 등록+분배 성공 품목도 즉시 매핑 학습한다.
- 서버 파싱 API는 AI 후보 300개 밖에 있는 학습 품목도 전체 품목 목록에서 찾아 적용한다.
- 거래처명은 공백 제거 후 정확/부분 매칭하고, 가장 가까운 길이의 거래처를 우선 선택한다.

## 기대 효과

- `화이트 3박스 취소`, `화이트 40박스 추가`처럼 수량과 액션이 달라도 기존 수동 매칭을 재사용한다.
- `주광농원`과 `주광`처럼 줄마다 표현이 달라지는 거래처명을 더 안정적으로 잡는다.
- 취소 줄에서 한 번 맞춘 품목도 다음 붙여넣기에서 자동 매칭된다.
