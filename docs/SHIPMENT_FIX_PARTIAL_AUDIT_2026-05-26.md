# 출고확정 부분확정 점검 (2026-05-26)

## 증상

견적서관리의 확정현황관리에서 `21-01`이 부분확정으로 표시되고, `콜롬비아 알스트로`가 미확정 상태인데 웹에서 확정 시도가 실패했다.

사용자 확인 기준으로 `nenova.exe`에서는 같은 차수/품목군 확정이 정상 동작했다.

## 원인

전산 `usp_ShipmentFix`는 운영 DB에서 `@CountryFlower` 파라미터를 지원하며 품목군 단위로 확정을 처리한다.

웹 `/api/shipment/fix`도 SP 자체는 `CountryFlower`별로 호출하도록 보완되어 있었지만, SP 호출 전에 웹 자체 음수재고 guard가 `21-01` 차수 전체 품목을 기준으로 먼저 실행되고 있었다.

따라서 같은 차수 안의 다른 품목군에 이슈가 있으면, 실제로 확정하려는 `콜롬비아 알스트로`가 미확정이고 SP 기준으로는 처리 가능한 상황이어도 웹에서 먼저 차단될 수 있었다.

## 보완

`usp_ShipmentFix`가 `@CountryFlower`를 지원하는 구조에서는 웹의 차수 전체 음수재고 사전 차단을 적용하지 않고, `nenova.exe`와 동일하게 SP의 품목군 단위 검증/확정 결과를 기준으로 삼는다.

`@CountryFlower`가 없는 구형 SP 구조에서는 기존처럼 차수 전체 음수재고 guard를 유지한다.

견적서관리 사전검증 모달도 음수재고 경고가 있다는 이유만으로 버튼을 비활성화하지 않도록 조정했다. 실제 확정 시도는 POST `/api/shipment/fix`를 통해 전산 SP 결과를 따른다.

## 충돌 방지 기준

- 부분확정 차수에서는 차수 전체 상태를 근거로 미확정 품목군 확정을 막지 않는다.
- 출고확정/취소는 직접 `isFix` 업데이트가 아니라 `usp_ShipmentFix` / `usp_ShipmentFixCancel`을 기준으로 유지한다.
- 웹 사전검증은 사용자 경고용이며, 실제 확정 성공/실패 판단은 운영 SP와 같은 범위여야 한다.
