# 영업매출관리 페이지 기획

작성일: 2026-06-03

## 목적

네노바웹에 `영업매출관리` 메뉴를 추가해 이카운트 판매현황 기준 업체별/차수별 매출을 2024년, 2025년, 2026년으로 비교한다.

핵심은 이카운트 원본 거래처명과 네노바에서 실제 통용하는 업체명이 다르기 때문에, 원본 매출을 바로 비교하지 않고 먼저 거래처 매칭/분배를 거친 뒤 비교표를 만든다는 점이다.

## 확인한 샘플 파일

### `25년 24차_양재동.xlsx`

- 시트: `판매현황`
- 구조:
  - 1행: 회사명/조회기간
  - 2행: 헤더
  - 컬럼: `일자-No.`, `거래처명`, `품목명`, `수량`, `단가(vat포함)`, `공급가액`, `부가세`, `합계`, `적요`
- 예시:
  - `아이엠 / Hydrangea White / 합계 14,891,800`
  - `소재2호 / Ruscus Green / 합계 17,807,900`
  - `(주)미카엘플라워 / Hydrangea White / 합계 1,935,000`
  - `선미원예(중매1484) / Hydrangea White / 합계 361,500`

### `매출비교.xlsx`

- 월별 시트: `1월`, `2월`, `3월`, `3월 (2)`, `4월`, `5월`, `6월`, `Sheet1`
- 구조:
  - 차수별 묶음 컬럼
  - 각 차수마다 `25년`, `26년`, `성장률`
  - 행은 네노바 통용 업체명 기준
- 예시 통용명:
  - `미우`, `소재2호`, `그린`, `꽃길`, `알파`, `꽃동산`, `레바논`, `미카엘`, `꿀벌`, `공주`, `성남`, `송우`, `선미`, `스타일`, `코벤트`

## 샘플 매칭 방향

| 이카운트 거래처명 | 네노바 통용명 후보 | 비고 |
| --- | --- | --- |
| `(주)미카엘플라워` | `미카엘` | 법인 표기 제거 |
| `선미원예(중매1484)` | `선미` | 중매번호/원예 제거 |
| `주식회사 송우플라워시스템` | `송우` | 법인/상호 긴 이름 축약 |
| `중매1536 (스타일)` | `스타일` | 괄호 안 통용명 우선 |
| `아이엠` | `미우` | 기존 대화 기준, `2층미우` 별칭과 연결 필요 |
| `그린화원` | `그린` | 통용명 축약 |
| `(주)플라워녹색공간` | `녹색` 후보 | 확인 필요 |
| `나래꽃(중매1390)` | `나래꽃` | 괄호 제거 |

| `주식회사 꿀벌원예` | `꿀벌` | 법인/원예 제거 |
| `중매1453호 유니온` | `유니온` | 중매번호 제거 |
| `코벤트원예 2호` | `코벤트` | 호수 표기 제거 |

자동 매칭은 후보만 제시하고, 확정 저장은 사용자가 선택한 뒤에만 한다.

## 메뉴 위치

우선안:

- 그룹: `채권관리`
- 메뉴명: `영업매출관리`
- 경로: `/sales/revenue-management`
- 표시 위치: `채권관리` 그룹 안에서 `판매현황` 바로 아래, `거래처별 채권` 바로 위

이유:

- 이카운트 판매현황 원본을 기준으로 매출 비교를 하는 화면이라 `/sales/status`, `/sales/ar`, `/sales/tax-invoice`, `/ecount/dashboard`와 같은 업무 흐름에 있다.
- 단순 통계가 아니라 업체 매출 원본과 매칭 상태를 관리하므로 `통계화면`보다는 `채권관리`가 더 자연스럽다.
- 실제 메뉴 화면 기준으로도 `판매현황`은 원본 매출 조회, `영업매출관리`는 연도/차수별 비교와 업체명 매칭, `거래처별 채권`은 채권 확인 순서가 되어 업무 흐름이 이어진다.

### 1. 상단 조건 영역

- 비교 연도: `2024`, `2025`, `2026`
- 조회 기준:
  - `이카운트 원본`
  - `네노바 매칭 후`
  - `미매칭 포함`
- 버튼:
  - `이카운트 엑셀 업로드`
  - `이카운트 API 불러오기`
  - `매칭 검증`
  - `비교표 생성`
  - `엑셀 다운로드`

### 2. 대시보드 탭

- 매출 감소 TOP 10

컬럼:

- 감소 업체
- 신규 업체
- 누락 업체

### 4. 이카운트 원본 탭

업로드 또는 API로 불러온 원본을 그대로 보여준다.

컬럼:

- 연도
- 차수
- 이카운트 일자-No.
- 이카운트 거래처명
- 매칭 통용명
- 품목명
- 수량
- 단가(vat포함)
- 공급가액

기능:

- 원본 합계와 매칭 후 합계 대조
- 공급가액+부가세=합계 검증
- 같은 거래처/품목 합산 보기
- 원본 이름별 그룹 보기

### 5. 업체명 매칭 탭

이 화면이 핵심이다.

좌측:

- 미매칭 이카운트 거래처명 목록
- 원본 합계
- 사용된 품목
- 출현 연도/차수

우측:

- 네노바 통용명 후보
- 네노바 거래처 후보
- 기존 주문등록 매칭 후보
- 기존 `customer-mappings.json` 후보
- 직접 입력 통용명

매칭 방식:

- `N:1`: 여러 이카운트 거래처명을 하나의 통용명으로 합산
  - 예: `그린화원`, `그린꽃`, `그린` → `그린`
- `1:1`: 이카운트 거래처명 하나를 통용명 하나로 연결
  - 예: `(주)미카엘플라워` → `미카엘`
- `1:N 분배`: 이카운트 거래처명 하나가 여러 내부 통용명으로 나뉘는 경우
  - 기준: 품목명, 적요, 비율, 수동 금액 배분 중 선택
  - 기본값은 금지. 사용자가 명시 저장한 분배룰만 적용

저장 규칙:

- 자동 후보는 저장하지 않는다.
- 사용자가 `확정 저장`을 누른 매핑만 다음 업로드부터 적용한다.
- 매핑 변경 시 이전 비교표는 재계산 로그를 남긴다.

### 6. 업체 상세 탭

업체 하나를 클릭하면 연도별 상세를 보여준다.

구성:

## 매칭 알고리즘

1. 원본 거래처명 정규화
   - `(주)`, `주식회사`, `원예`, 괄호, 중매번호, 공백, 특수문자 제거
2. 기존 확정 매핑 우선 적용
3. 주문등록 거래처 매핑 후보 적용
   - 기존 `customer-mappings.json`와 동일한 정규화 규칙 활용
4. 네노바 `Customer` 이름/비고/주문코드 후보 검색
5. 통용명 사전 후보 검색
6. 후보가 1개면 `후보확정대기`
7. 후보가 여러 개면 `수동확인필요`
8. 후보가 없으면 `미매칭`

## 검증 기준

업로드/조회 후 반드시 아래를 보여준다.

- 이카운트 원본 총합
- 매칭 후 총합
- 미매칭 총합
- 분배룰 적용 총합
- 원본 총합과 최종 비교표 총합 차이
- 공급가액+부가세와 합계 차이
- 같은 원본 거래처명이 다른 통용명으로 나뉜 건
- 같은 통용명이 여러 원본 거래처명을 합친 건

최종 비교표 생성 조건:

- 미매칭 금액이 있으면 경고 표시
- 사용자가 `미매칭 제외하고 보기`를 선택할 수는 있지만, 합계 카드에는 제외 금액을 항상 표시
- 매칭 변경 후 자동 재계산

## 이카운트 API 사용 기준

처음 구현은 엑셀 업로드 기반으로 시작한다.

이유:

- 이카운트 API 호출 횟수 제한이 있다.
- 현재 샘플도 이카운트 판매현황 엑셀 기준이다.
- 페이지 구조와 매칭룰 검증 전에는 API 조회를 반복하지 않는 것이 안전하다.

이후 API 연동:

- 사용자가 연도/차수/기간을 선택하고 `이카운트 API 불러오기`를 누를 때만 호출
- 같은 연도/차수는 캐시된 Batch가 있으면 재사용
- 강제 재조회는 별도 버튼으로 분리
- 이 기능은 이카운트 데이터를 조회만 하며 이카운트에 쓰기/변경하지 않는다.

## 구현 단계

### 1단계: 엑셀 업로드 + 매칭 + 비교표

- `/sales/revenue-management` 페이지 추가
- `영업매출관리` 메뉴 추가
- 이카운트 판매현황 엑셀 업로드
- `매출비교.xlsx` 형태의 차수별 비교표 생성
- 미매칭/매칭 후보 UI
- 확정 매핑 저장
- 엑셀 다운로드

### 2단계: DB 저장 및 이력 관리

- Batch/Raw/Mapping/Summary 저장
- 매핑 변경 이력
- 업로드 파일별 원본 보존
- 같은 차수 재업로드 시 기존 Batch 비교

### 3단계: 이카운트 API 조회

- API 조회는 수동 버튼으로만 실행
- 조회 결과를 Batch로 저장
- API 호출 횟수와 마지막 조회시간 표시

### 4단계: 네노바 판매/출고 데이터와 대조

- 네노바 확정 출고 매출과 이카운트 판매현황 차이 표시
- 차이 원인:
  - 미전송
  - 업체명 매칭 차이
  - 금액/부가세 차이
  - 차수 귀속 차이

## 작업 전 주의

- 이 기능은 조회/분석/매칭 기능이다.
- 이카운트 원본 데이터를 수정하거나 이카운트에 판매전표를 전송하지 않는다.
- 자동 매칭은 후보 제시까지만 하고, 확정 저장은 사용자 승인 후에만 한다.
- 금액 비교는 `합계`와 `공급가액+부가세`를 모두 보되, 실무 비교표 기본값은 이카운트 `합계` 기준으로 둔다.
- 기존 `/sales/status`, `/sales/ar`, `/ecount/dashboard`와 중복되는 새 기능을 만들지 않고, `영업매출관리`는 연도/차수/업체명 매칭 비교에 집중한다.
