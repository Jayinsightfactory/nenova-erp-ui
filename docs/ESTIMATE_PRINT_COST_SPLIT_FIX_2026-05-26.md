# 견적서 출력 단가별 분리 수정 (2026-05-26)

## 문제

견적서 출력에서 같은 품목이 여러 차수에 걸쳐 있고 차수별 단가가 다를 때도 한 줄로 합산되었다.

이 경우 합친 수량과 하나의 단가가 같이 표시되어, 실제 차수별 단가가 다른 구조를 숨기는 문제가 있었다.

## 원인

`pages/estimate.js`의 출력 집계 키가 `EstimateType + ProdKey`만 사용했다.

따라서 같은 품목이면 단가가 달라도 같은 그룹으로 묶였다.

## 수정

출력 집계 키를 다음 기준으로 확장했다.

- `EstimateType`
- `ProdKey` 또는 `ProdName`
- `Unit`
- `Cost`

이제 같은 품목이라도 단가가 다르면 별도 행으로 출력된다. 같은 품목, 같은 단위, 같은 단가인 경우만 수량과 금액이 합산된다.

## 주의

차수별 단가가 다른 것은 정상 업무 케이스이므로, 단가가 다른 행을 억지로 평균 단가처럼 합치면 안 된다.
