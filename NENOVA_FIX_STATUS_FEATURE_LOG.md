# 차수 확정 현황 / 구간 확정취소 구현 로그

작성일: 2026-05-15

## 구현 목적

15차 견적이나 입고를 수정하려면 현재 확정된 후속 차수까지 모두 풀어야 한다. 기존에는 한 차수씩 확정취소해야 했고, 후속 차수가 확정되어 있으면 단일 취소가 막히는 흐름이었다.

이번 작업은 다음을 목표로 한다.

- 차수별 확정/미확정/부분확정 상태를 한 화면에서 확인
- 선택한 시작~끝 차수 구간을 높은 차수부터 낮은 차수 순서로 일괄 확정취소
- 확정 시 `전재고 + 입고 - 출고 < 0` 때문에 실패할 품목을 미리 표시

## 추가된 화면

`/shipment/fix-status`

메뉴 위치:
- 데스크톱: 입/출고관리 → 차수 확정 현황
- 모바일 메뉴: 입/출고관리 → 차수 확정 현황

화면 기능:
- 시작 차수 / 끝 차수 조회
- 차수별 상태 표시: 확정, 부분확정, 미확정, 출고없음
- 출고라인, 확정라인, 미확정라인, 카테고리 수 표시
- 음수재고 예상 품목 표시
- 선택 구간 확정취소 실행

## 추가된 API

`GET /api/shipment/fix-status`

Query:
- `fromWeek`
- `toWeek`

응답:
- `weeks`: 차수별 확정 현황
- `negative`: 확정 시 음수재고가 예상되는 품목 목록

`POST /api/shipment/fix-status`

Body:
- `fromWeek`
- `toWeek`
- `force`

동작:
- 선택 구간 안의 확정된 `ShipmentDetail`을 찾는다.
- `CountryFlower` 단위로 전산 SP `usp_ShipmentFixCancel`을 호출한다.
- 높은 차수부터 낮은 차수 순서로 처리한다.
- 선택 구간 이후에 이미 확정된 차수가 있으면 기본적으로 한 번 경고하고, 사용자가 확인하면 `force=true`로 진행한다.

## 기존 확정 검증 보강

수정 파일:
- `pages/api/shipment/fix.js`

개선 내용:
- `2026-15-01`처럼 연도 포함 차수로 들어와도 `OrderWeek = 15-01`로 정규화해 검증한다.
- 음수재고 검증 SQL을 `OrderYear + REPLACE(OrderWeek, '-', '')` 비교 기준으로 바꿨다.
- 확정 전 검증 모달에서 부족 품목의 전재고/입고/출고/부족 수량이 더 안정적으로 표시된다.
